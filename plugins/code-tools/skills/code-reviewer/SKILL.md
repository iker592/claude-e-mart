description: Reviews code for best practices, potential bugs, and improvements
---

When reviewing code, analyze for:

1. **Code Quality**
   - Clear naming conventions
   - Proper code organization
   - DRY principles

2. **Potential Bugs**
   - Edge cases not handled
   - Null/undefined checks
   - Race conditions

3. **Security**
   - Input validation
   - SQL injection risks
   - XSS vulnerabilities

4. **Performance**
   - Unnecessary loops
   - Memory leaks
   - N+1 queries

Provide specific, actionable feedback with code examples when suggesting improvements.
