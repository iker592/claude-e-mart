---
description: Commit changes, push to remote, and create a pull request
---

Follow these steps:

1. Run `git status` to see all changes
2. Run `git diff` to review the changes
3. Stage all relevant changes with `git add`
4. Create a commit with a clear, descriptive message
5. Push to the remote branch
6. Create a pull request using `gh pr create`

Ask the user for the PR title and description if not provided in "$ARGUMENTS".
